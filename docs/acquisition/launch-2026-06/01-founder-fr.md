## Version courte

Je lance Embir en public.

L’idée est simple : arrêter de traiter la rencontre comme une file de profils à swiper, et repartir de ce que les gens cherchent vraiment. Amour, amitié, sport, événements, fun : Embir classe la découverte autour d’intentions déclarées, de préférences explicites et de pertinence réciproque.

C’est encore tôt. Le produit se construit ville par ville, avec une surface publique volontairement prudente : pas de faux compteurs, pas de promesse de densité locale, pas de storytelling artificiel.

## Version longue

Je travaille sur Embir, une application de rencontre construite autour d’une question plus saine que “qui est le plus visible dans le feed ?” :

Qu’est-ce que chacun cherche, et est-ce que cette intention est réciproquement pertinente ?

Le lancement est volontairement progressif. Embir met en avant :

- des intentions déclarées dès le départ ;
- une logique de préférences réciproques ;
- un tunnel de découverte avant inscription ;
- des aperçus anonymisés quand des profils publics réels existent ;
- un positionnement mobile-first.

Je préfère être transparent : Embir est jeune. Le but maintenant n’est pas de prétendre qu’une communauté massive existe déjà partout, mais de construire une base propre, honnête et compréhensible.
